## 2. 마케팅 전략 방향

### 랜딩 페이지의 핵심 포지셔닝

브레인시티 안에서 병원, 학교, 공원, 상업시설을 생활 반경 안에 둔 실거주 중심 대단지 아파트로 포지셔닝합니다. 투자성만 앞세우기보다 “의료·교육·녹지·교통이 함께 모이는 미래 생활권”을 전면에 두는 것이 적합합니다.

### 참조 페이지 반영 방향

참조 페이지 `https://m-bee.kr/braincity`는 첫 화면에서 `선착순 동호지정`, `방문예약`, `관심고객 등록`, `하단 고정 전화 CTA`, `E모델하우스`, `방문 희망일시 입력`을 전면에 배치해 전환 압박을 강하게 주고 있습니다. 이 기획서에서는 그 전환 방식은 적극 반영하되, 사업 규모와 세부 사실은 PDF 기준으로 유지합니다.

반영할 요소:

- 히어로 상단에 `선착순 동호지정` 또는 `잔여세대 상담` 성격의 즉시 행동 문구 배치
- `방문예약`과 `관심고객 등록`을 분리한 2트랙 CTA 구성
- 모바일 하단 고정바에 대표번호 `010-6689-2348` 직접 노출
- 중단부에 `E모델하우스` 또는 `평면 미리보기` 섹션 추가
- 상담 폼에 `방문 희망일시` 선택 필드 추가
- `상담하기` 클릭 시 간단한 사전 입력 후 전화 연결되는 프리콜 레이어 구성

반영하지 않을 요소:

- 참조 페이지의 사업 규모 표기 중 PDF와 충돌하는 수치
- 검증되지 않은 문의 수, 긴급성 숫자, 과도한 마감 문구
- 공개 근거가 확인되지 않은 표현의 직접 복제

### 상담하기 버튼 동작 원칙

`상담하기` 버튼은 바로 전화 앱으로 넘기지 않고, 먼저 짧은 프리콜 입력 레이어를 띄운 뒤 `이름`, `연락처`를 남기면 `010-6689-2348`로 전화 연결되도록 설계합니다. 이렇게 해야 상담사가 통화 이탈이나 부재중 전화 발생 시 다시 연결할 수 있고, 사용자는 전화 연결 전 본인 확인 절차를 짧게 이해할 수 있습니다.

개발 및 DB 식별 기준:

- 랜딩 페이지 개발자 지정값: `이정임`
- 고객 DB 저장 시 `developer_name` 컬럼에 항상 `이정임` 기록
- 전화상담, 방문예약, 관심고객 등록 등 모든 수집 폼에 동일하게 적용

권장 흐름:

1. 사용자가 `상담하기` 클릭
2. 하단 시트 또는 팝업에 이름, 연락처 입력
3. 짧은 안내 문구와 개인정보 동의 체크
4. `전화 연결하기` 버튼 클릭
5. `tel:01066892348`로 전화 앱 연결

주의:

- 웹페이지 단독으로는 사용자가 입력만 하고 완전히 자동으로 발신까지 진행시키는 것은 불가능하므로, 최종 단계는 사용자의 전화 연결 클릭이 필요
- 입력값 저장은 앱스크립트 웹앱 또는 CRM 연동이 있어야 실제 활용 가능

### 운영 담당 기준

- 개발자명: `이정임`
- 모든 리드 데이터에 `developer_name=이정임` 자동 기록
- 사용자는 개발자정보를 직접 입력하지 않음

### 폼별 저장 이벤트

- `상담하기`: 이름, 연락처 입력 후 앱스크립트 저장, 이후 전화 연결
- `방문예약`: 이름, 연락처, 관심 타입, 방문 희망일시 저장
- `관심고객 등록`: 이름, 연락처 중심 최소 저장
- `분양자료 받기`: 이름, 연락처 저장 후 자료 제공 또는 콜백 연계

### 구현 메모

- 프런트엔드에는 숨김 필드 또는 상수값으로 `developer_name: "이정임"` 포함
- 앱스크립트에서도 한 번 더 `developer_name` 기본값을 보정
- 시트 구조는 단일 시트를 우선 사용하고, 추후 운영 규모가 커지면 리드 유형별 시트 분리 가능
- 전화 연결 버튼은 DB 저장 성공 여부와 무관하게 사용자가 직접 누를 수 있게 하되, 저장 실패 시 재시도 메시지를 별도 노출
